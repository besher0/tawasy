# Cake Production System Structure

## 1. Database Structure (ER Diagram Concept)

### Users Table
- id (PK)
- name
- phone
- role (Admin, Factory Manager, Shop Manager, Shop Employee)
- shop_id (FK, nullable for factory)

### Shops Table
- id (PK)
- name
- location
- contact_info

### Orders Table (Custom Cakes)
- id (PK)
- shop_id (FK)
- customer_name
- customer_phone
- delivery_datetime
- total_price
- deposit_amount
- payment_status (Unpaid, Partial, Paid)
- status (New, Reviewing, In_Production, Ready, Delivered, Cancelled)
- is_urgent (Boolean)
- notes

### Order_Items Table (Multiple cakes per order)
- id (PK)
- order_id (FK)
- cake_type (Cake, Dummy, etc.)
- layers (int)
- shape
- filling
- special_details
- people_count
- reference_images (JSON/Array of URLs)

### Daily_Essentials Table (Next Day Orders)
- id (PK)
- shop_id (FK)
- category (Ready_Cake, Raw_Materials, Pieces, Supplies)
- item_name
- quantity
- notes
- status (Pending, Prepared, Out_for_Delivery)
- target_date

---

## 2. User Journey (UX Flow)

### Shop Employee Journey:
1. Login -> Dashboard.
2. Click "New Order" -> Fill details + Upload photos -> Submit.
3. Monitor Order Status via Dashboard notifications.
4. "Next Day Orders" tab -> Add daily supplies needed for tomorrow.

### Factory Manager Journey:
1. Login -> Kanban Board view.
2. Filter by "Urgent" or "Shop".
3. Change status as production moves (Reviewing -> Preparing -> Ready).
4. Print Production Sheets for the kitchen staff.
5. Review Statistics for daily performance.

---

## 3. Production Optimization Suggestions
- **Color Coding:** Use Red for Urgent, Orange for < 3 hours to delivery, Green for Ready.
- **Batched Prep:** Group orders by "Filling Type" or "Cake Shape" to save time in the kitchen.
- **Digital Station:** Tablets in the kitchen showing the Kanban board so chefs can update status in real-time.
